# Paper C BMC MRM reproducibility code - PSR + delta* sensitivity

This folder is a self-contained repository to run the Paper C simulation pipeline on a VM (e.g., GCP). If this folder is uploaded to GitHub, the contents of this folder should be placed at the repository root.

## 1) One-time setup (R packages)

```bash
cd <repository-root>
Rscript scripts/00_install_deps.R
```

## 2) Quick check (small run)

On macOS/Linux or Git Bash:

```bash
bash scripts/98_quickcheck.sh
# outputs under quickcheck_paperc/
```

On Windows/RStudio, use the RStudio Console:

```r
source("scripts/98_quickcheck.R")
# outputs under quickcheck_paperc/
```

## 3) Full GCP run with tmux

```bash
chmod +x scripts/99_run_paperc_tmux_gcp.sh
bash scripts/99_run_paperc_tmux_gcp.sh
tmux ls
```

The tmux launcher defines three sessions:
- `pc_main`: full grid, IPCW + DR(glm) (main paper)
- `pc_methods`: subset with ML/TMLE (supplement)
- `pc_sens`: heavier sensitivity (optional; supplement)

Outputs are written under the corresponding `OUT_DIR` folders. Figure scripts generate reproducible figure files under `OUT_DIR/figures/`; the submitted BMC MRM figure PNGs are publication-format exports of these generated figures. Figure 3 requires `SAVE_SENS_CURVES=1`, which is enabled in the quickcheck and methods runs. Figure 4 is generated by `scripts/24_make_paperc_method_compare_figures.R` from the method-comparison summaries; the quickcheck runs this script as a smoke test, while the manuscript figure should be regenerated from the full methods-subset run.

Important: the quickcheck outputs are smoke-test figures generated from a reduced grid. They are not expected to match the submitted manuscript figures in panel structure or numerical values. The submitted Figure 1 is reproduced from the full main run with `MIS_SPEC_LEVELS=0,1`. The submitted Figure 4 is reproduced from the full methods-subset run with `MIS_SPEC_LEVELS=0,1`, `DELTA_DGM_LEVELS=D1_zero,D2_constant,D3_late_only`, `SCENARIO_SUBSET=S01,S09,S18`, and `RUN_TMLE=1`.


## 4) Implementation vignette (MIMIC-IV demo, illustration only)

This repository includes an optional script to reproduce an implementation-vignette table for piecewise SPD(t) using the MIMIC-IV Clinical Database Demo (v2.2).
The script writes only aggregate summaries (no patient-level output).

### Run (from the repo root)

If you have the demo zip:

```bash
OUT_DIR=output_paperc_mimic_demo \
MIMIC_ZIP=/absolute/path/to/mimic-iv-clinical-database-demo-2.2.zip \
TAU_HOURS=72 \
DO_PIECEWISE=1 \
Rscript scripts/26_run_mimic_demo_worked_example.R
```

Outputs:
- `output_paperc_mimic_demo/tables/eTable6_MIMIC_demo_piecewise_SPDt.csv`
- `output_paperc_mimic_demo/tables/eTable6_MIMIC_demo_piecewise_SPDt_notes.txt`

Notes:
- This is for implementation and reporting illustration only, not clinical inference.
- You must comply with PhysioNet credentialing and the MIMIC-IV data use agreement.

## v13 final note

The quickcheck includes explicit output validation. It fails if the expected raw tables, summary tables, and Figures 1-4 are not created or if the residual-sensitivity curves contain no finite logRR values. The virtual sensitivity engine was also made more robust across R versions by using `delete.response()` and dropping the Cox-model intercept column, which prevents counterfactual linear predictors from becoming NA in small Windows/RStudio quickcheck runs. These changes affect code robustness only and do not change the manuscript estimands or full analysis specification.


## v13 fix

The Figure 1 script now calls `data.table::fifelse()` explicitly. This avoids a Windows/RStudio failure in `scripts/23_make_paperc_figures.R` when `data.table` is available but not attached to the search path. This is a namespace fix only; it does not change the analysis specification.
